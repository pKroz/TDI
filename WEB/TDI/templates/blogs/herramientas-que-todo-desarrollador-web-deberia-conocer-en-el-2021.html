Ingeniería web

No ha pasado mucho tiempo desde que los desarrolladores web y el consecuente desarrollo de proyectos basados en tecnologías web se ha ido popularizando considerablemente. Hemos podido reconocer el nacimiento de nuevas tecnologías y herramientas espectaculares como React o la preferencia de aplicar los diseños responsive, por otro lado, aún seguimos viendo a nuestros viejos compañeros de desarrollo como jQuery que no han parado y han seguido evolucionando y mejorando con el paso del tiempo.

Actualmente podemos tener acceso a cientos de bibliotecas, frameworks y conjuntos de herramientas que nos ayudan a ser más productivos en nuestro flujo de trabajo y así poder obtener resultados en tiempos muchos más óptimos y con un menor esfuerzo de lo que se estimaba hace algún tiempo atrás. Incluso tenemos la posibilidad de colaborar de forma remota en proyectos con gran facilidad gracias a los nuevos sistemas de control de versiones sorprendentemente eficiente, hasta los navegadores nos proporcionan soluciones y herramientas que amplían todavía más las posibilidades de desarrollo.

Podemos tomarnos todo el día y enumerar cada una de las herramientas y es que hay tantas posibilidades y opciones, pues sabemos que en el mundo de desarrollo y programación el aprendizaje es constante y en muchas ocasiones puede ser abrumador el mantenerse al día. Pero, ¡No temas!, para facilitarte el trabajo hoy te hemos seleccionado las mejores herramientas que debes usar como desarrollador web. A continuación, te presentamos una lista:

  • Visual Studio Code
  • Chrome Developers Tools
  • jQuery
  • Git
  • npm
  • TypeScript
  • Bootstrap
  • Angular
  • Sass

Visual Studio Code

Arrancamos por lo más obvio: un editor de código fuente. Por años Sublime Text estuvo reinando y a día de hoy sigue siendo uno de los editores de código más usados, hasta que en 2014 apareció Atom con su propuesta basada en Electron. Microsoft reconoció el potencial de Atom y decidió seguir el mismo camino lanzando Visual Studio Code, también basado en Electron para implementar Chromium y Node.js, pero sin usar el editor Atom: emplea Monaco, el mismo de Visual Studio Team Services. También trae Git integrado, herramientas de debug increíblemente útiles, una amplia variedad de extensiones y plugins.

Chrome Developers Tools

Casi siempre deseamos poder editar HTML y CSS en tiempo real o depurar JavaScript mientras analizamos el comportamiento de nuestro sitio web o proyecto, pero o bien hemos de recurrir a módulos de Node, o hemos de montar un entorno de desarrollo específicamente para ese propósito. Con Chrome Developer Tools nada de eso es necesario, pues se trata de un conjunto de herramientas para desarrolladores que se encuentran integradas en todos los navegadores Chrome que nos permiten mejorar los tiempos de carga y a conseguir una mejor optimización de nuestro proyecto.

jQuery

JavaScript es por excelencia un lenguaje de programación front-end pero a día de hoy sigue presentando unos pequeños dolores de cabeza: incompatibilidades en ciertos navegadores, sintaxis complejas, y un consumo de recursos en ocasiones elevado. Todo esto cambió a partir del 2006 con la aparición de jQuery, una pequeña librería JavaScript que mejora notablemente todas sus funcionalidades. jQuery es rápido y práctico, pero sobre todo es multiplataforma, diseñado para simplificar la programación front-end de cualquier proyecto. Más de 10 años después sigue siendo una de las librerías de JavaScript más populares, hasta el punto que incluso WordPress empezó a implementarlo como un estándar de su infraestructura con la llegada de Gutenberg.

Git

Imagínate realizar cambios importantes en tu proyecto y que por A o B pierdas todo tu progreso: nunca empieces un proyecto nuevo sin usar un repositorio Git primero. Si eres nuevo en Git (no confundir con GitHub) nunca es tarde para iniciarse por primera vez especialmente si vas a trabajar con Visual Studio Code: podremos aprovechar al máximo su integración con el control de versiones y de repositorios. Y realmente Git es sencillo de usar una vez lo tengamos instalado en nuestro equipo.

npm

npm es útil para construir cosas fascinantes a través de herramientas esenciales de JavaScript. Tiene la capacidad de gestionar equipos, no es necesario configurar, y proporciona auditorías de seguridad. En ese sentido para empresas es capaz de ofrecer características de seguridad, desarrollos sin duplicar, soporte y un control de acceso tremendamente útil. Entre sus características nos permite instalar una infinidad de paquetes de código abierto con una amplía biblioteca donde hay miles de expansiones públicas y de todo tipo por descubrir. Gracias a su soporte recibiremos advertencias automáticas si nos topamos ante cualquier código inseguro, por lo cual no hay mucho peligro de que acabemos instalando un módulo que contenga software peligroso para nuestro equipo.

TypeScript

TypeScript se trata de un superconjunto escrito a partir de JavaScript, en otras palabras, nos compilará JavaScript en un código más simple. Es compatible con cualquier navegador, con cualquier host y con todos los sistemas operativos, y entre sus posibilidades tiene una buena convergencia con JavaScript nativo: podemos seguir trabajando con nuestro código JavaScript normal y llamar a TypeScript dentro de él.

Bootstrap

Si lo que buscamos es la mayor eficiencia y rapidez a la hora de desarrollar proyectos puramente front-end, o tal vez agilizar el front-end para priorizar el back-end, entonces Bootstrap te será muy útil. Básicamente Bootstrap nos permitirá estructurar toda la maquetación de tu proyecto mediante elementos comunes y reutilizables reduciendo drásticamente las horas de trabajo que dediquemos a nuestro proyecto.

Angular

Desarrollado por Google y lanzado en 2011, Angular nos permite ampliar la sintaxis de HTML en un entorno mucho más expresivo y fácilmente legible. Nos ayuda a desarrollar más rápido y a diseñar aspectos que de otra manera no podríamos construir en HTML básico.

Sass

SASS es un preprocesador de CSS que nos permitirá escribir nuestros estilos CSS con un código mucho más compacto, eficiente y corto, sin repetir en exceso una y otra vez las mismas reglas y estilos. Es un proyecto de código abierto que arrancó en 2006 y acabó por redefinir completamente la manera en la que empleamos CSS moderno para estilizar nuestros proyectos. Al comienzo puede ser un poco complejo de comprender, pero una vez le tomemos el gusto será difícil no vivir sin el uso de las variables, los mixins o de anidar estilos en bloques. Además, las hojas de estilos de SASS ofrecen una mejor legibilidad de nuestros estilos, y aún más importante, compila directamente en CSS comprimido.
